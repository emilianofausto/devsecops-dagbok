DevSecOps Diary API Documentation

This document describes the REST API for the DevSecOps Diary application. The API provides endpoints for Create, Read, Update, and Delete (CRUD) operations on diary entries.

Base URL

/api/entries

Data Schemas

DiaryEntryCreate

Used for creating a new diary entry.

FieldTypeValidationDescription
titleStringRequired, Length: 1-100Title of the entry.
contentStringRequired, Min Length: 1Main content of the entry.
categoryStringRequired, Length: 1-50Category of the entry.

DiaryEntryUpdate

Used for modifying an existing diary entry.

FieldTypeValidationDescription
titleStringOptional, Length: 1-100Title of the entry.
contentStringOptional, Min Length: 1Main content of the entry.
categoryStringOptional, Length: 1-50Category of the entry.

DiaryEntryResponse

Returned by the API after retrieving or modifying an entry.

FieldTypeDescription
idIntegerUnique identifier for the entry.
titleStringTitle of the entry.
contentStringMain content of the entry.
categoryStringCategory of the entry.
created_atDatetime (ISO 8601)UTC timestamp of entry creation.

Endpoints

GET /api/entries

Retrieves a list of all diary entries.

  • Success Response: 200 OK — Returns List[DiaryEntryResponse]

GET /api/entries/{entry_id}

Retrieves a specific diary entry by its ID.

  • Path Parameters: entry_id (integer, required)
  • Success Response: 200 OK — Returns DiaryEntryResponse
  • Error Response: 404 Not Found — {"detail": "Entry with ID {entry_id} not found"}

POST /api/entries

Creates a new diary entry.

  • Request Body: DiaryEntryCreate (application/json)
  • Success Response: 201 Created — Returns DiaryEntryResponse

PUT /api/entries/{entry_id}

Updates an existing diary entry. Only provided fields will be modified.

  • Path Parameters: entry_id (integer, required)
  • Request Body: DiaryEntryUpdate (application/json)
  • Success Response: 200 OK — Returns DiaryEntryResponse
  • Error Responses:
    • 400 Bad Request — {"detail": "Payload contains no valid modifications"}
    • 404 Not Found — {"detail": "Entry with ID {entry_id} not found"}

DELETE /api/entries/{entry_id}

Deletes a specific diary entry from the database.

  • Path Parameters: entry_id (integer, required)
  • Success Response: 200 OK — {"message": "Entry {entry_id} successfully deleted"}
  • Error Response: 404 Not Found — {"detail": "Entry with ID {entry_id} not found"}
